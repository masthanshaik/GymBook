# 🚀 Deploy GymBook to Render (with your own domain)

This deploys 3 things: the **API** (FastAPI), the **website** (React), and a
**PostgreSQL database** — all on Render's free tier.

---

## STEP 0 — Put your code on GitHub (Render deploys from a repo)

If it's not on GitHub yet:
```bash
cd ~/Downloads/gym_platform_final
git init
git add .
git commit -m "GymBook"
```
Then create an empty repo on github.com and push:
```bash
git remote add origin https://github.com/masthanshaik/GymBook.git
git branch -M main
git push -u origin main
```

---

## STEP 1 — Create the Blueprint on Render

    1. Go to https://render.com → sign up / log in (free)
2. Click **New** → **Blueprint**
3. Connect your GitHub and pick the `gymbook` repo
4. Render reads `render.yaml` and shows 3 services:
   `gymbook-api`, `gymbook-web`, `gymbook-db`
5. Click **Apply** — it starts building. The database + API come up first.
   (First build takes ~5–10 min. Free database is created automatically.)

---

## STEP 2 — Connect the two URLs to each other

After the first deploy, Render gives you two URLs like:
- API:  `https://gymbook-api.onrender.com`
- Web:  `https://gymbook-web.onrender.com`

Now tell each one about the other:

**a) Point the website at the API**
- Render dashboard → **gymbook-web** → **Environment**
- Set `VITE_API_URL` = `https://gymbook-api.onrender.com/api/v1`
- Save → it redeploys the site

**b) Allow the website through the API's CORS**
- Render dashboard → **gymbook-api** → **Environment**
- Set `FRONTEND_ORIGINS` = `https://gymbook-web.onrender.com`
  (later add your custom domain too, comma-separated)
- Save → API restarts

---

## STEP 3 — Seed real starting data (optional)

To create your owner login on the live database, open
**gymbook-api** → **Shell** in Render and run:
```bash
python3 seed_data.py
```
This creates demo@gymbook.com / Demo@1234. **For real customers**, log in
once, then create your real gym via Sign Up and delete the demo later.

---

## STEP 4 — Add your custom domain

For the **website** (what customers visit, e.g. app.yourgym.com):
1. Render → **gymbook-web** → **Settings** → **Custom Domains** → **Add**
2. Enter your domain (e.g. `app.yourgym.com`)
3. Render shows a **CNAME** record. Go to your domain registrar's DNS settings
   and add it:
   - Type: `CNAME`  ·  Name: `app`  ·  Value: `gymbook-web.onrender.com`
4. Wait for DNS (5 min–1 hr). Render auto-issues a free HTTPS certificate.

Then update CORS to include it:
- **gymbook-api** → Environment → `FRONTEND_ORIGINS` =
  `https://gymbook-web.onrender.com,https://app.yourgym.com`

(Optional: give the API its own subdomain like `api.yourgym.com` the same way,
then update the website's `VITE_API_URL` to match.)

---

## ✅ Done
Visit your domain → the styled GymBook loads → log in and run your gym.

---

## ⚠️ Important notes for real customers
- **Free tier sleeps** after 15 min idle — first visit then takes ~30s to wake.
  For a real gym, upgrade gymbook-api + gymbook-db to the **Starter plan
  (~$7/mo each)** so they stay always-on and data is retained long-term.
- **Free database expires after 90 days** on Render. The paid plan removes this.
  Back up your data regularly (Render dashboard → database → Backups).
- **SECRET_KEY** is auto-generated by Render (secure). Don't share it.
- **Webcam photos** work automatically once you're on HTTPS (your domain).

## 🆘 If a build fails
- Check the build logs in the Render dashboard for the failing service.
- API fails on "psycopg2" → already handled (postgres:// auto-converted).
- Web shows unstyled → ensure build command includes `npm run build:css`
  (it's in render.yaml already).
